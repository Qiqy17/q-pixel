# OpenAI 生图接入设计

## 目标

让 Q像素的“AI灵感”直接调用 OpenAI GPT Image 2 生成图片，并自动进入现有的导入、网格校准、像素化和色号处理流程。

## 边界

- 浏览器和桌面资源包不保存或传输 OpenAI API key。
- 本机同步服务新增 `/api/ai/generate`，从 `OPENAI_API_KEY` 环境变量读取密钥。
- 服务端调用 `gpt-image-2` 的 `/v1/images/generations`，返回内嵌 PNG data URL。
- 前端把返回图片当作一次新的图片导入，目标网格尺寸使用用户填写的宽高。
- 生成提示词强调清晰轮廓、纯色块、有限色板、无抗锯齿和无纹理，以便后续还原成拼豆图纸。
- 未配置密钥、额度不足、接口错误和返回格式异常都显示可理解的中文提示，不暴露密钥或原始敏感请求头。

## 验收标准

1. 未设置 `OPENAI_API_KEY` 时，界面显示“未配置 OpenAI API Key”，不会发起外部请求。
2. 设置密钥后，`/api/ai/generate` 可以返回 `imageDataUrl`，前端自动生成指定网格的 Q像素图纸。
3. 生成后的图纸默认开启原图对比层，并保留当前的高保真/均衡/易制作导入模式。
4. 8765 电脑同步服务和 8766 平板 HTTPS 服务行为一致。
5. 自动化测试覆盖请求校验、无密钥错误、图片 data URL 导入和现有前端回归。
