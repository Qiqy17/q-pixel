# Q像素导入、还原与编辑安全优化设计

## 目标

减少导入图片后的试错成本，避免还原优化破坏细节，并在跨设备编辑时明确提示未保存内容。

## 功能范围

### 导入信息可见化

- 导入方式弹窗增加当前模式的说明和预估信息区域。
- 生成图纸后显示“导入摘要”：模式、图纸尺寸、当前色号数、是否使用校准、原图对比状态。
- 导入完成后默认保留原图对比能力，用户可以直接关闭或调节透明度。

### 非破坏式多档还原

- 将单一“原图还原”入口改为三档：轻度、均衡、强细节。
- 每次还原先保存当前图纸快照，可使用现有撤销恢复。
- 轻度优先去除孤立杂色，均衡兼顾轮廓连续性，强细节尽量保留高光和小面积颜色。
- 连续点击同一档不会重复处理同一结果；切换档位从当前图纸继续，并显示实际色号变化。

### 编辑安全

- 维护当前图纸是否有未保存修改的状态。
- 保存成功后清除状态，绘制、导入、优化、恢复等改变图纸的操作会标记为未保存。
- 返回首页、打开其他项目、切换设计前，如果有未保存修改，弹出保存/继续/取消选择。
- 不拦截浏览器刷新，避免在测试环境和嵌入式应用中重复弹窗；页面标题和保存按钮提供轻量状态提示。

### 响应式检查

- 编辑页在 390px、768px、1440px 视口不出现横向溢出。
- 工具栏按钮可换行，关键操作不被固定面板遮挡。

## 兼容与错误处理

- 不修改项目文件和同步服务数据结构。
- 现有旧项目没有安全状态字段时默认视为已保存。
- 导入或还原失败时不改变未保存状态，不清空当前画布。
- 保存失败保留未保存状态，并提示已经保留在当前设备或页面缓存。

## 验收标准

- 三种还原档位均有独立可测试策略，并能撤销。
- 导入摘要在直接导入和校准导入后都能更新。
- 未保存项目离开编辑页会得到明确选择，取消后仍留在当前画布。
- 仓库网页和桌面版资源测试均为 `PASS`，移动端无横向溢出。
