# 材质分类与颜色预设设计

## 目标

在不增加操作复杂度的前提下，解决材质数量增加后下拉列表过长、材质颜色选择单一的问题。材质纹理仍由现有真实位图和光影渲染管线负责，新增功能只负责选择和组合。

## 交互方案

采用“分类 + 色彩预设”的两级选择面板：

1. 材质分类：石材、木材、布料、纸张、玻璃、金属、塑料、摄影棚、印刷等。
2. 分类内材质：只展示当前分类的材质按钮，保留现有材质值和兼容性。
3. 色彩预设：提供暖色、冷色、自然、深色、低饱和、浅色等预设色板，并保留自定义颜色输入。
4. 当前选择摘要：显示“材质名称 + 色彩名称”，让用户知道当前组合。
5. 选中色彩只改变底图的色调叠加，不覆盖真实纹理、凹凸、粗糙度和光影。

## 数据与兼容性

- 继续使用现有 `materialBackgroundSelect` 的材质 value，保存的设计文件无需迁移。
- 新增颜色预设状态并纳入样式预设保存/恢复。
- 旧设计加载时没有颜色预设则使用当前背景色和默认色彩预设。
- 自定义背景图片仍保持独立入口，不与材质色板混用。

## UI 范围

- 将长下拉框替换为紧凑的分类标签、材质按钮和色板区域。
- 桌面端采用左右布局；窄屏自动改为上下布局。
- 保留现有光照、边框、装饰、导出按钮，不新增重复的“材质预览”入口。
- 材质按钮只显示名称和简短质感标识，不为每一项加载大图，避免界面卡顿。

## 验证标准

- 现有材质全部仍可选择，旧设计文件可正常恢复。
- 色彩预设切换后，纹理细节和光影仍可见，不能退化为纯色块。
- 样式预览和下载图片使用同一颜色组合。
- 桌面、电脑网页、平板网页布局不出现横向溢出。
- `node --check web/app.js`、`git diff --check` 和浏览器回归页通过。
