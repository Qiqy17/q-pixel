# Q像素项目历史版本界面设计

## 目标

让用户可以从设计文件操作面板查看当前项目的历史快照、打开历史快照进行预览，以及将某个历史快照恢复为当前版本，同时保留当前版本以便再次回退。

## 交互范围

- 设计文件操作面板增加“历史版本”入口；没有历史快照时按钮禁用。
- 历史版本弹窗按保存时间从新到旧展示最多 12 条现有快照，每条显示标题、保存时间、画布尺寸，并提供“预览打开”和“恢复为当前版本”。
- “预览打开”把快照载入画布，但清空当前项目 ID并追加“历史预览”标题，不改变原项目，避免用户不知情覆盖原文件。
- “恢复为当前版本”先把当前内容写入历史，再把选中的快照写回同一个项目 ID并同步远端。恢复后历史列表仍可继续使用。
- 弹窗可通过关闭按钮或遮罩关闭，移动端保持可滚动且按钮不超出视口。

## 数据与错误处理

- 复用已有 `project.history`、`normalizeProjectHistory`、`withProjectHistory` 数据结构，不新增服务端接口。
- 历史快照损坏或不存在时提示用户并保持当前画布不变。
- 远端只有摘要而没有项目正文时，预览和恢复先按现有项目加载逻辑请求正文；请求失败不覆盖当前内容。
- 恢复保存失败时保留本地恢复结果，并提示“已恢复到当前设备，远端同步暂不可用”。

## 测试标准

- 保留现有历史快照、同步合并和冲突副本测试。
- 增加恢复构造测试：恢复后使用原项目 ID、当前版本进入历史、选中快照内容成为当前正文。
- 增加界面结构检查：历史入口和历史弹窗存在、无重复 ID、无历史时入口禁用。
- 在开发版和桌面应用资源版分别运行浏览器测试，并检查桌面资源与仓库文件一致。
